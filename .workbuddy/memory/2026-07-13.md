### 第二轮（18:49 日志）：GatherWriterMetadata 卡住轮询不退出
- **根因**：`WaitForAsyncOperation` 检查了错误的变量。
  - `IVssAsync::QueryStatus(&hrResult)` 返回 `S_OK` (0x00000000) 表示查询调用本身成功。
  - 异步操作的实际状态在输出参数 `hrResult` 中。
  - VSS 异步常量：`VSS_S_ASYNC_PENDING`=0x00042309, `VSS_S_ASYNC_FINISHED`=0x0004230A, `VSS_S_ASYNC_CANCELLED`=0x0004230B。
  - 代码第 1049 行检查 `hrQuery == VSS_S_ASYNC_FINISHED`（即 `S_OK == 0x0004230A`），永远为 false。
  - 应该检查 `hrResult == VSS_S_ASYNC_FINISHED`。
- `vssadmin` 不受影响，因为它使用 `IVssAsync::Wait()` 或正确检查 `pHrResult`。
- 代码注释（第 1047 行）"S_OK / VSS_S_ASYNC_PENDING 均视为仍在进行中"体现了对 API 的误解。
